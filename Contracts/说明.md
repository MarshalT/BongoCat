# EOS链上宠物猫合约文档

## 1. 合约概述

BongoCat是一款基于EOS区块链的数字宠物猫游戏，允许用户收集、繁殖和交易独特的虚拟猫咪。每只猫都拥有不同的基因，决定了其外观和特性。本合约实现了猫咪的创建、拥有权管理、繁殖、交易和市场功能。

## 2. 合约功能

- **猫咪铸造**：包括初代猫咪NFT铸造和繁殖系统
- **猫咪所有权**：转移、赠送或交易猫咪
- **繁殖系统**：基于基因组合算法
- **市场功能**：挂单、购买、拍卖
- **猫咪互动**：喂食、训练、升级等
- **特殊道具**：提升猫咪属性的消耗品

## 3. 数据结构

### 3.1 猫咪结构
```cpp
struct cat {
    uint64_t id;              // 唯一标识符
    name owner;               // 所有者账户
    uint64_t genes;           // 基因编码，决定外观和特性
    uint32_t birth_time;      // 出生时间
    uint32_t cooldown_end;    // 繁殖冷却期结束时间
    uint16_t generation;      // 代数
    uint16_t cooldown_index;  // 决定繁殖冷却期长度
    uint64_t parent1_id;      // 父猫ID
    uint64_t parent2_id;      // 母猫ID
    
    // 猫咪属性
    uint8_t stamina;          // 耐力
    uint8_t agility;          // 敏捷
    uint8_t cuteness;         // 可爱度
    uint8_t level;            // 等级
    
    uint64_t primary_key() const { return id; }
    uint64_t get_owner() const { return owner.value; }
    uint64_t get_genes() const { return genes; }
    uint8_t get_level() const { return level; }
    uint32_t get_birth_time() const { return birth_time; }
    uint16_t get_generation() const { return generation; }
    bool is_ready_to_breed() const { return cooldown_end <= current_time_point().sec_since_epoch(); }
};
```

## 4. 合约接口

### 4.1 主要操作接口

| 接口名称 | 描述 | 参数 | 权限 |
|---------|------|------|------|
| mint | 铸造初代猫咪（用户为自己铸造，每个账户限mint一只） | 无 | 调用者账户 |
| transfer | 转移猫咪所有权 | name from, name to, uint64_t cat_id, string memo | from账户 |
| breed | 繁殖猫咪 | uint64_t parent1_id, uint64_t parent2_id | 两只猫的所有者 |
| createsale | 创建售卖 | uint64_t cat_id, asset price | 猫咪所有者 |
| cancelsale | 取消售卖 | uint64_t sale_id | 售卖创建者 |
| buysale | 购买猫咪 | uint64_t sale_id | 购买者 |
| createauction | 创建拍卖 | uint64_t cat_id, asset start_price, uint32_t duration | 猫咪所有者 |
| placebid | 参与拍卖出价 | uint64_t auction_id, asset bid_amount | 竞拍者 |
| endauction | 结束拍卖 | uint64_t auction_id | 任何账户 |
| feed | 喂食猫咪 | uint64_t cat_id | 猫咪所有者 |
| train | 训练猫咪 | uint64_t cat_id, uint8_t attribute_type | 猫咪所有者 |
| upgrade | 升级猫咪等级 | uint64_t cat_id | 猫咪所有者 |
| buyitem | 购买道具 | uint64_t item_id, uint8_t quantity | 购买者 |
| useitem | 使用道具 | uint64_t cat_id, uint64_t item_id | 猫咪和道具所有者 |

### 4.2 铸造机制说明

- 初始总量限制为10000只猫咪
- 每个账户只能为自己铸造一只初代猫咪，通过`minters_table`记录已铸造账户
- 铸造费用初始为10 EOS代币，通过调用合约时转账实现
- 每铸造1000只猫咪，价格上涨10%（base_price * 1.1）
- 猫咪的基因编码由合约随机生成，使用区块哈希和交易ID作为熵源
- 最高等级为100，升级需要支付BCAT代币（EOS 链上代币 合约没有代币）

## 5. 繁殖机制

繁殖系统是BongoCat合约的核心功能，基于以下规则：

1. **基因混合**：父母双方的基因会通过特殊算法组合，产生具有独特特征的子代猫咪
2. **冷却期**：繁殖后的猫咪进入冷却期，冷却期随着繁殖次数增加而延长
   - 首次繁殖：1小时冷却期
   - 第二次繁殖：2小时冷却期
   - 第三次繁殖：4小时冷却期
   - 随后每次翻倍，最长不超过7天
3. **代际限制**：高代数猫咪繁殖会增加特殊特征的出现几率
4. **繁殖费用**：每次繁殖需支付5 EOS代币作为费用

基因编码使用64位整数，按不同位段划分为多种特征：

- 毛色：基因的第0-7位
- 眼睛：基因的第8-15位
- 花纹：基因的第16-23位
- 耳朵：基因的第24-31位
- 尾巴：基因的第32-39位
- 特殊特征：基因的第40-63位

## 6. 用户互动流程

### 6.1 新用户入门
1. 创建EOS账户
2. 铸造一只初代猫咪或从市场购买现有猫咪
3. 喂食、训练猫咪提升属性
4. 寻找另一只猫咪进行繁殖

### 6.2 繁殖流程
1. 选择两只属于同一用户的猫咪
2. 确保两只猫咪均不在冷却期
3. 调用`breed`函数进行繁殖
4. 支付繁殖费用
5. 获得新的子代猫咪

### 6.3 交易流程
1. 调用`createsale`设置猫咪价格
2. 其他用户使用`buysale`购买猫咪
3. 交易完成后自动转移所有权

### 6.4 拍卖流程
1. 猫咪所有者调用`createauction`创建拍卖
2. 设置起拍价和拍卖时长
3. 其他用户使用`placebid`参与竞拍
4. 拍卖结束时，最高出价者获得猫咪，原所有者获得拍卖金额

## 7. 权限与安全

- 每个操作只能由相应权限的账户执行，如猫咪所有者、售卖创建者等
- 所有转账交易通过EOS的权限系统确认
- 所有操作需要active权限或更高权限
- 合约实现了重入攻击防护
- 随机数生成使用区块哈希、交易ID和时间戳的组合，增加预测难度

## 9. 示例调用

### 铸造猫咪
```bash
# 发送10 EOS给合约以铸造猫咪
cleos push action eosio.token transfer '["alice", "bongocat", "10.0000 EOS", "mint"]' -p alice@active
# 触发mint操作
cleos push action bongocat mint '[]' -p alice@active
```

### 转移猫咪
```bash
cleos push action bongocat transfer '["alice", "bob", 1, "礼物"]' -p alice@active
```

### 繁殖猫咪
```bash
# 发送5 EOS给合约以支付繁殖费用
cleos push action eosio.token transfer '["alice", "bongocat", "5.0000 EOS", "breed"]' -p alice@active
# 触发breed操作
cleos push action bongocat breed '[1, 2]' -p alice@active
```

### 创建销售
```bash
cleos push action bongocat createsale '[1, "100.0000 EOS"]' -p alice@active
```

### 创建拍卖
```bash
cleos push action bongocat createauction '[1, "50.0000 EOS", 86400]' -p alice@active
```

### 竞拍出价
```bash
# 发送出价金额给合约
cleos push action eosio.token transfer '["bob", "bongocat", "60.0000 EOS", "bid:1"]' -p bob@active
# 触发placebid操作
cleos push action bongocat placebid '[1, "60.0000 EOS"]' -p bob@active
```

### 升级猫咪
```bash
# 发送BCAT代币给合约以支付升级费用
cleos push action bcat.token transfer '["alice", "bongocat", "5.0000 BCAT", "upgrade:1"]' -p alice@active
# 触发upgrade操作
cleos push action bongocat upgrade '[1]' -p alice@active
```
